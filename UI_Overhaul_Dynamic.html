Interface Modified

This mod is specially tailored towards 1920 × 1080 and 2560 × 1440, with no UI scaling usage. 3840 × 2160 users can go up to 1.5 on the UI scaling, but no more.
Wider screens will have some small adaptations to them, like a wider outliner. Smaller resolutions than 1920 × 1080 will require UI scaling below 1.
Check the supported resolutions thread to confirm your settings.
Using any UI scaling outside of the table is freely up to you, but you'll have to life with the consequences of an overflowing interface.


Pretty much everything visible has had a change, but the short gist of it;

  • Adapted UI to resize itself depending on user resolution. See supported resolutions for more info.
  • Most windows have increased height and width, to increase space for relevant content.
  • Increased the height of most list elements.
  • Fixes and optimizes vanilla UI wherever possible.
  • Bigger Empire Creation.
  • New Planet Viewer (See below for info).
  • New Ship Designer that vastly increases the whole window space to allow for more ship sections, like New Ship Classes.
  • New Fleet Manager that hopefully increases effectiveness
  • In-game usable Tech Tree, made by: turanar. (Using the built-in web browser)
  • Added in keyboard icons from Immersive Keyboard Icons.

Alternative Design

Planet Viewer

The planet viewer was recently changed, and as I know people hate change, there are alternatives:

  • Bigger Planet View (Will retain the old design)
  • Bigger Planet View - Building Box Dirty Fix (If you want more than 36 buildings slots)
  • Planetary Diversity - Planet View (Vanilla look, with larger cityscape) New portraits messes this up.

Use only one of them.

Ship Designer

Ship designer now has a fully horizontal style, so if you want to retain the old vertical one, subscribe to this alternate sub-mob:

  • Old Ship Designer


Load Order

I'd recommend putting this mod at the very bottom, as in last in your load order.
This guarantees proper UI changes, as the launcher loads the mods from top to bottom.
Only exception is with specific instructions given with compatibility patches, or if you want a specific UI mod to overrule the changes in this one.

It should be fine as long as your vertical resolution matches a resolution in the table.
