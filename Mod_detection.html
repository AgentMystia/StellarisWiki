Mod detection

There are three methods to detect the presence of another mod, and they require that the target mod has set up ways to become detectable.

Global Flag: You can set a global flag to identify your mod in an event that will fire at game start. The main disadvantage is that you can't check for it before game start, making it unusable for things like policies or buildings for example.

These two other methods are preferable:

ID Trigger: An ID Trigger is a scripted trigger that is set to be always true. The mod that wants to detect it will need to use a placeholder trigger set to be always false. If the target mod is present, it will overwrite this trigger.

ID Variable: This method is efficient in jobs calculations. The target mod creates a scripted variable set to 1, and to detect it, a placeholder variable set to 0 will be overwritten if the target mod is present.

It can then be used as a multiplier, or checked for its value.

Detection Methods
Method Target mod Detection
Global Flag set_global_flag = my_mod_active has_global_flag = my_mod_active
ID Trigger my_mod_active = { always = yes } my_mod_active = yes
ID Variable @my_mod_active = 1 - Multiply the amount of jobs by the scripted variable

- Check value with check_variable_arithmetic
