Dualcolor ethic icons are hard to look at

Is it okay if we undo changes that added dual color ethic icons? They're really hard to look at, it looks like something made for those blue-red 3D glasses. I feel that usually there isn't any ambiguity from just using the icon for the regular form of the ethic to refer to both, and the space saved from having to put the regular and the fanatic icon in a few places is absolutely not worth it. -- PointlessSerpent (talk) 13:29, 17 May 2024 (UTC)

I feel that in most cases, they are helpful to indicate that both regular and fanatic icons meet a particular requirement. In my view it is definitely preferable to listing near-identical icons next to each other, and not including the fanatic icon at all would feel insufficient at this point, like there's some info missing. -- Lorenerd11 (talk) 13:38, 17 May 2024 (UTC)
I don't really see how not including the fanatic icon would be a problem, my understanding is that the regular icon applies to both variants unless it's explicitly stated otherwise, and we could even have some sort of "icon interpretation" page that explicitly clarifies that and other icons if that isn't clear. I also don't see why having two icons next to each other is a problem if you think that's better, the icons are easy to read and understand, and they're very small so page space isn't really an issue. I'm happy to help edit pages to make the iconography more clear, but the split icons are just really hard on my eyes. Job, civic, and tech icon links

I've been wondering about the use of redirects in these icons. Right now, they link to redirect pages, which then lead to the main pages.

This is something that has largely been retired for planet modifier, building, and leader trait icons ever since they started to use templates.

The redirect pages look neater aesthetically in the url preview, but they create some problems. For instance, if a civic icon is used on the Civics page, it links to the redirect page of the civic that then leads back to the main page, when it would be more useful as an anchor instead link. This could be fixed with the {{ifpage}} template, where if the user is on a specific page, the link would create an anchor link instead of a redirect, but this would have to be manually added in every single icon subtemplate, which is itself a tedious process, while potentially making the icon links themselves a bit too complicated.

Another problem is that in some cases, the redirect pages do not exist, resulting in dead-end links. I've addressed that in {{Civic}} and {{Job}} templates, where I used #ifexist to check whether a redirect page exists, and if it does not, it will just lead to the Jobs and Civics pages directly. That said, the #ifexist function is a bit resource-expensive, so its use should be limited when not necessary. I've already updated the civic and job templates to skip its use when on civic or job pages.

All of this is making me wonder if the redirect pages are worth it in the end, though. - Lorenerd11 (talk) 23:28, 15 September 2024 (UTC)
