Overview

Modding operations is similar to modding Archaeological Sites, since they use the same dice roll mechanic and pop-up window. Operations are defined at Stellaris/common/espionage_operation_types/*.txt

Data Structure

Property Description
target = <target_key> Target type for the operation. Valid target types are 'country', 'megastructure', 'starbase', 'fleet', 'army', 'pop_faction', 'spynetwork', 'federation' and 'none'[1].
target_potential = { <conditions> } A block of Conditions to determine the list of available targets for the operation. Has the same scope as defined in target property.
potential = { <conditions> } A block of Conditions to determine if this operation can be executed. If evaluated to false, it will not appear on UI.
allow = { <conditions> } A block of Conditions to determine if this operation can be executed. If evaluated to false, it will be greyed out and locked for selection.
abort = { <conditions> } A block of Conditions to determine if this operation should be aborted. If evaluated to true, operation will be cancelled immediately[needs confirmation].
categories = { <category_key> } A list of categories for operation. See Categories.
desc = <localisation_key> OR
desc = {
trigger= { <conditions> }
text = { <localisation_key> }
}
 Description for the operation. If only localisation key is specified, decription will always be the same. If conditional version is used, multiple blocks can be added (different descriptions for different conditions)[2].
picture = <sprite_key> A reference to a GFX instance (sprite key) that indicates the picture of this operation.
resources = { <economy_unit> } An Economy Unit to determine the category, launch cost and resource upkeep of this Operation.
spy_power_cost = < int > Infiltration level required for operation
stages = <int> Amount of stages in this operation. Should match number of defined stages below.
stage = {
difficulty = <value>
icon = <string>
event = <string>
}
 Stage definition, order dependent. Amount of stage blocks should match number of defined stages above.

diffculty - base difficulty for this stage, icon - GFX icon reference, event - event to fire when finished the state.

on_roll_failed = { <effects> } Effects to fire when a roll fails, with scope this = spy operation. Should at least include the following effect, responsible for processing dice rolls and adding insights. Common RANDOM_EVENTS value is operation_random_events_generic
standard_espionage_operation_on_roll_failed = {
	RANDOM_EVENTS = <scripted effect>
}
on_create = { <effects> } Effects to fire upon operation creation, with scope this = spy operation.

Categories

There are 5 primary and 5 secondary categories of operations. Primary category defines the UI icon/filter in espionage menu. Purpose of secondary categories is unclear. There can be only one primary category.[3]

Primary category must be defined before secondary categories.

Primary categories:

Category key Name
op_cat_manipulation Manipulation
op_cat_sabotage Sabotage
op_cat_subterfuge Subterfuge
op_cat_provocation Provocation
op_cat_infiltration Infiltration

Secondary categories:

Category key Name
op_cat_government Government
op_cat_diplomacy Diplomacy
op_cat_economy Economy
op_cat_technology Technology
op_cat_military Military

Example

Below is the example of definiton for "Gather Information" operation. More examples could be found at Stellaris/common/espionage_operation_types/*.txt

operation_gather_information = {
	target = none
	categories = { op_cat_subterfuge }
	picture = GFX_evt_spymaster
	desc = {
		trigger = { has_nemesis = yes }
		text = operation_gather_information_desc
	}
	desc = {
		trigger = { has_nemesis = no }
		text = operation_gather_information_no_dlc_desc
	}
	stages = 1

	resources = {
		category = operations
		cost = {
			influence = 20
		}
		upkeep = {
			energy = 4
		}
	}

	spy_power_cost = 10
	potential = {
		target = { is_primitive = no }
	}
	allow = {
		custom_tooltip = {
			is_running_espionage_operation = no
			fail_text = operation_one_at_a_time
		}
	}
	stage = {
		difficulty = @diff_t0
		icon = GFX_espionage_chapter_icon_document
		event = operation.7000
	}

	on_roll_failed = {
		standard_espionage_operation_on_roll_failed = {
			RANDOM_EVENTS = operation_random_events_low_stakes
		}
	}
	on_create = {
		target = {
			set_timed_country_flag = {
				flag = espionage_operation_targeted_by@root.owner #Used in random Spy Network events
				days = @operationTargetedByTimer
			}
		}
	}
}

Espionage Events

Modding events for espionage operations is similar to modding any other event and is described in Event modding article. However, there are several moments that should be mentioned:

  1. Event must have espionage_operation_event type.
  2. Event has additional property espionage_operation, which can be either yes or no.[4]
espionage_operation_event = {
   espionage_operation = yes
   <other event properties>       
}

Examples of espionage events could be found at Stellaris/events/nemesis_espionage_events.txt

Notes

  1. ↑ CWT documentation says that none equals to "spy network target": However, the Nemesis events seem to treat that scope as the country that has been selected as target for the operation. Scopes for country, spynetwork and none require further in-game testing.
  2. ↑ What would happen if several desc blocks has conditions evaluated to true at the same time?
  3. ↑ Needs confirmation
  4. ↑ So far it seems that setting it to yes makes it appear in the operation window with the toast in the top left, while setting it to no makes it appear as a separate popup. Any other differences need closer inspection.
