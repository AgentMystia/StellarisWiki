Placeholders

Placeholders are for compatibility with other mods, are bits of code that are designed to be ignored by the game until they are overwritten by another mod. This allows to reduce the amount of errors in the logs. Not all entries are capable of having placeholders without breaking the code, either because they are subject to the DUPL bug, too complex to handle, or don't allow them. The file naming must respect the load order to be loaded first, for more details, see Overwriting specific elements.

Since v3.14, inline scripts have been established as good practices, which can replace the old global flags and trigger placeholder concept (if they can be used at the specific points; initiated by modder TTFTCUTS (Gigastructures)[1] and RoAdd independently), in combination with global scripted variables. This has the advance of increased performance and loading time, and reduces even the error logs to zero. So instead of checking a condition every time the evaluation is performed, this happens once when the game is loaded, depending on whether a mod is missing (placeholder value 0 when merging) or present (value 1 provided by the mod). No extra scripted effect or triggers are necessary anymore, just the global scripted "placeholder variable" for the specific mod and if needed the standardized inline script (e.g. merger_of_rules/toggled_code from "The Merger of Rules" mod).

If you just want other mods to be compatible with your mod, all you need to do is declare a specific scripted mod variable. For example: 

@has_planetary_diversity = 1

If you want your mod to be compatible with other mods, you must replace the previous mod condition (if present) and insert the related code in the inline_script parameter. For example: 

    inline_script = {
        script = merger_of_rules/toggled_code
        code = "
            custom_tooltip = {
                fail_text = legendary_leader_planet_no_terraform
                this = {
                    NOT = { has_planet_flag = legendary_leader_planet }
                }
            }
        "
        toggle = @has_planetary_diversity
    }

If several other mods support the specific code, you can add all the mod variables to the toggle parameter. For example: 

        # Ethics and Civics Alternative - FunEFork / Ethics & Civics: Bug Branch
        inline_script = {
            script = merger_of_rules/toggled_code
            code = "
                has_valid_civic = civic_slaving_despots
                has_valid_civic = civic_slaving_despots_corpo
            "
            toggle = @[ has_eca_funefork + has_bug_branch ] # This is an OR toggle
        }
Note
Scripted variables are FIOS, e.g. The merger of Rules uses ~~~~z_check_variables.txt for all mod IDs.

Known issues
Authorities are loaded in a random order since v3.12.
District and megastructures order is dependant of load/declaration order.
Technologies will inherit the potential part of the code if the overwriting tech doesn't have one, thus the use of an ID Trigger: a scripted trigger that the mod author sets to always = yes while the mod checking it uses a placeholder one.
