Data Structure

  • weight_modifier - Modifies how likely should a leader have this Agenda.
  • modifier - A block of Modifiers to be applied to the Empire for as long as this leader remain on the ruler's seat.

Whenever a leader becomes the ruler or a candidate of an Empire with an Authority that has has_agenda = yes, if the leader doesn't have an Agenda, a random Agenda is chosen for the leader and they will preserve the Agenda for their lifetime and will never change to another.

Unlike Mandates, the Agendas don't have some sort of on_term_started. All an Agenda can be relevant to the game is its Modifiers.

There are neither Conditions to check the Agenda nor Effects to change the Agenda.

(However, it is still possible for modders to nearly ensure that a desired custom agenda is selected, as shown in the second example below.)

Example of Default Agenda

Science Leap

agenda_science = {
	weight_modifier = {
		weight = 100
		modifier = {
			factor = 1.5
				from = {
					has_ethic = ethic_materialist
				}
			}
		modifier = {
			factor = 2
				from = {
					has_ethic = ethic_fanatic_materialist
				}
			}
		modifier = {
			factor = 1.5
			leader_class = scientist
		}
	}
	
	modifier = {
		all_technology_research_speed = 0.10
	}
}

Example of Modded, Near-Guaranteed Agenda

Modded Agenda

agenda_modded_agenda = {
	weight_modifier = {
		weight = 100000000      #Such a high weight means this agenda will almost certainly be selected, unless ruled out by the modifier below.
		modifier = {
			factor = 0.0        #This modifier section will reduce the chance of getting this agenda to zero.
			from = {
				NOT = {
    					has_civic = civic_custom_civic     #Thus, any empire that does not have this civic has zero chance of getting this agenda, while any empire with this civic probably will.
				}
			}
		}
	}

	modifier = {
		all_technology_research_speed = 0.05
		pop_happiness = 0.05
		species_leader_exp_gain = 0.10
	}
}

In this latter example, each new leader (including the leader at game start) is very likely to be assigned this agenda, for as long as the empire retains the key custom civic.

Graphics files for Agendas

Some of the base game graphics in gfx/interface/icons/agendas are 124x124, others are 129x129.

Making your new agenda icons at 124x124 is fine and also makes centering content a bit easier as the x/y dimensions are even.

Color your image from an existing agenda graphic in your preferred editor. One way to do this is in GIMP you can use the colorize tool on your new graphic and use the dropper from the dialogue on an existing graphic; from there tweak the hue and lightness since there may be variance.

Make sure your finished flattened layer is the full size of the canvas to avoid blurring or other odd distortions.

Export the graphic using AGBR8 with DXT5 compression.

Define the sprite

The game will expect a spriteType definition with a name matching the pattern GFX_council_agenda_icon_<agenda_key>

If your agenda key is defined as agenda_modded_agenda (as in the above example) then your spriteType will look like this: 

spriteType = {
    name = "GFX_council_agenda_icon_agenda_modded_agenda"
    texturefile = "gfx/interface/icons/agenda_icons/agenda_modded_agenda.dds"
}

This definition can go in any .gfx file in your interface/ folder (not to be confused with gfx/interface).

The texturefile (dds) filename does not have to follow any pattern but prefixing it with agenda_ is a good way to stay organized.

Troubleshooting agenda graphics

Issue: Graphic does not appear in game

Solution: Check your error log to see that you wrote your spriteType definition correctly. There will be an error log saying Missing sprite for council agenda


Issue: Color is weird/off

Solution: Check you exported with AGBR8 and not RGBA8 for example. Might need to carefully tweak your colors a bit more as sometimes even slight variances can be not obvious in the editor but obvious in game.
