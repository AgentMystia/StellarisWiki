Customizable icon link and hovertext

I feel that the {{icon}} template could use a way to set custom link or hovertext, particularly when the same icon is used for a different purpose (e.g. event jobs), or when you want the icon to link to elsewhere on the page (such as making job icons on the job article link to an anchor on the page rather than a redirect back into the jobs page).

This could be achieved by changing the code of the icon template from:
{{icon/{{padleft:|1|{{lc:{{{1}}}}}}}|{{lc:{{{1}}}}}|w{{#if:{{{2|}}}||NULL}}={{{2}}}}}
to:
{{icon/{{padleft:|1|{{lc:{{{1}}}}}}}|{{lc:{{{1}}}}}|w{{#if:{{{2|}}}||NULL}}={{{2}}}<!--
-->|hover{{#if:{{{hover|{{{3|}}}}}}||NULL}}={{{hover|{{{3}}}}}}<!--
-->|link{{#if:{{{link|}}}||NULL}}={{{link}}}<!--
-->}}
(added line breaks for improved readability)

All of the icons in the sub-templates would need to be edited to accommodate this too, but adding this to the core template would allow the functionality to be used in the first place. -- Lorenerd11 (talk) 08:20, 7 May 2024 (UTC)

Icon sub-categories

Mildtables do not support certain row-wide style parameters

I noticed this while trying to add emphasized dividing lines between rows; you cannot apply custom border-top to mildtable rows. 

{| class="mildtable"
|- style="border-top:6px double #DDD"
| Sample 1 || Sample 2
|- style="border-top:6px double #DDD"
| Sample 3 || Sample 4
|}
Sample 1 Sample 2
Sample 3 Sample 4

It works just fine when applied individually to each instead of the entire row: 

{| class="mildtable"
| style="border-top:6px double #DDD" | Sample 1
| style="border-top:6px double #DDD" | Sample 2
|- 
| style="border-top:6px double #DDD" | Sample 3
| style="border-top:6px double #DDD" | Sample 4
|}
Sample 1 Sample 2
Sample 3 Sample 4

This would however be very impractical to use on actual articles, so until this is fixed, border-top is practically unusable in mildtables right now. - Lorenerd11 (talk) 10:26, 6 October 2024 (UTC)
